Which Europe Gets Recommended?

An intra-European audit dashboard for ranked cultural visibility.

Catalogue diversity is not the same as ranked visibility. This dashboard follows the full audit story from data reality to model rankings, feedback-loop drift, mitigation trade-offs and governance reporting.

Research storyline

The dashboard answers a governance question, not a benchmark question.

The red thread is explicit: access is not visibility; visibility is model-dependent; repeated visibility can drift; mitigation must report its cost.

Main research question: how do recommender systems transform European catalogue diversity into ranked and repeated visibility across countries and languages?

Current evidence is shown only where the local outputs support it. Granular country, Spain-specific and religion/theme claims are marked as evidence gaps unless reliable support tables exist.

    Documentation

    How the work is structured and how to rerun it.

    This section is written for reviewers and group handoff.

    Research question

    Do movie recommender systems trained on real interaction data under-expose European, non-English and long-tail films in Top-20 recommendations, and can multimodal features or governance-aware re-ranking reduce the gap without substantially harming recommendation utility?

    Metrics

    Utility: NDCG@20, Recall@20, MAP@20 and Coverage@20. Cultural prominence: discounted group exposure, Prominence Gap and PACPG.

    Reproducibility commands

    uv venv --python 3.12 .venv
uv pip install --python .venv/bin/python -r requirements.txt
jupyter lab

    Deliverables

    Executed notebooks, HTML exports, model dashboard, caveat report and citation ledger. Raw data is intentionally excluded from distributable outputs.

    Expected outcome

    The project does not prove that recommenders can be built; it audits what they make visible.

    This ending clarifies the contribution for the proposal discussion: established recommender research is the baseline, while the project adds cultural-prominence governance.

    Technical outcome

    A reproducible, real-data pipeline that joins M3L-20M interactions, MovieLens identifiers, MPNet/CLIP features and Wikidata cultural metadata, then evaluates Top-20 recommendations with utility, exposure, coverage and PACPG metrics.

    Governance outcome

    A transparent audit dashboard that makes language differences, production-country proxies, co-productions, US-company involvement and metadata missingness visible beside model metrics.

    Research positioning

    Public recommender-system research profiles such as Markus Schedl's work show that recommendation models are already a mature technical field. Our proposal therefore focuses on cultural visibility, accountability and mitigation rather than on novelty for its own sake.

    Source: Markus Schedl profile, cited in the project ledger.

    Presentation takeaway

    The final result is an evidence-based answer to a governance question: when a platform has European and non-English works in the catalogue, do its rankings actually let users see them, and what ranking-utility cost follows from making them more prominent?
